AR Viewer User Guide

Loading a model

An existing (obj or glTF) model can be loaded into the AR viewer through an online depository.
An example repository containing a limited range of files can be found here.

The "Load model" tab in the menu defines a marker and a model within a specific directory.
When "import model" is clicked, the defined model is displayed on the defined marker (unless the file does not exist).
When "remove model" is clicked, the defined model is removed from the defined marker.
When "clear marker" is clicked, all models on the defined marker are removed. Only one instance of a marker can be used at any one time.

The directory from which a model is loaded is explained in Changing sources.

The scale of a model can be specified using a numerical value. Entering a single number will proportionately scale the model, and entering three numbers separated by spaces (eg "1 2 3") will scale x, y, and z independently.

Selected models can be manipulated using the mouse, touch screen or controller buttons in the menu. The reset button will return all models on the selected marker to their original position, orientation and scale on the marker.

Only models specified in the text boxes can be manipulated. If the top checkbox is ticked, all models on the defined marker are affected by manipulations. If it is unchecked, only the specified model is affected. By ticking the checkbox beside the reset button, all models are locked and cannot be manipulated via mouse or touch screen. The selected models can be manipulated via the on-screen controls as normal.



Changing sources

For ease of access, only the name of a model or marker is needed in the "Load model" tab. A specific naming convention is used to enable this to work. By default, a sample model repository is used.

The "Change source" tab in the menu allows the user to define a custom location for importing marker patterns and obj or gltf models. It should be noted that only online directories will be recognised. If a non-existent directory is called, no model will be loaded until a valid address is given. The default directory can be restored in each case by clicking "revert".

The viewer is capable of loading .obj and .gltf models. By clicking the corresponding file type button, the viewer will switch to load that model type. By default the viewer will load .obj files.

Marker file convention

Markers are read by the program from .patt files. .patt files are text files the AR Viewer program interprets in recognising real-world markers. The directory to a file location is defined using the format:
  https://example_directory.com/ Marker .patt
where "Marker" is the marker defined in Loading a model.

Creating markers

Markers are simple symbols framed by a black square which function as real-world targets to place virtual models on. Users can design their own symbols and use this site to generate a marker and corresponding .patt file. It is recommended that the default frame thickness (50%) is used to produce reliable markers. Alternatively, several example markers with their pattern files are available here.

Model file convention

Object models are read by the program from .obj files for shape, and .mtl files for colour. The directory to a file location is defined using the format:
  https://example_directory.com/ Model .obj and
  https://example_directory.com/ Model .mtl respectively.
Both files are defined by the same "Model" name in Loading a model. This requires the files to be located in the same directory with the same name. Objects loaded without material files will be displayed monochromatically.

Any files supplementing .gltf files should be located relatively as defined within them.
